CSS Basics

Basic Syntax:

CSS is made up of various rules. Each rule is comprised of a selector and a semicolon-separated list of declarations. Each declaration is made up of a property-value pair.

Selectors:

Selectors refer to the HTML elements to which CSS rules apply. There are many types of selectors but here are the most common.

  • Universal Selector: The universal selector selects elements of any type. You use * {} to call this.
  • Type Selector: Will select all elements of the given Element type. The name of the element type followed by {} is used to acll this
  • Class Selector: Will select all elements of the given Class, which is just an attribute that you place on a HTML element. .class-name followed by {} is used to call this
    • Syntax for class selectors: a period immediately followed by the case-sensitive value of the class attribute. Classes are not required to be specific to a particular element, so you can use the same class on as many elements as you want.
    • Another thing you can do with the class attribute is to add multiple classes to a single element as a space-separated list. Since whitespace is used to separate class names like this, you should never use spaces for multi-worded names and should use a hyphen instead.
  • ID Selector: Will select an element with the given ID, which is another attribute you place on an HTML element.
    • Unlike a class, an element can only have 1 ID.
    • An ID cannot be repeated on a page and should not contain any whitespace.
    • ID's use the # symbol immediately followed by the case-sensitive value of the ID
    • ID are often overused when a class would work better and should be used sparingly. They are best used when taking advantage of specificity or having links redirect to a section on the current page.
  • Grouping Selector: We can group selectors as a comma separated list to cut down on the repetition
  • Chaining selectors: You can also chain Class Selectors or a Class and ID selector to format very specific elements.

Descendant Combinator:

Combinators allow us to combine multiple selectors differently than either grouping or chaining them, they show a relationship between the selctors. There are 4 types of combinators in total, but we will cover 1 for now.

Descendant Combinator

- This is represented in CSS by a single space between selectors. This combinator will only cause elements that match the last selector to be selected if they also have and ancestor (parent, grandparent, etc.) that matches the previous selector.

Properties to get started with

There are some CSS properties that you're going to be using all the time, or at the very least more often than not.

  • color - will set an element's text color
  • background-color - will set the background color of an element

    • These color properies will accept several types of values.

    • Keyword - such as "red" or "transparent"
    • HEX - such as "#1100ff"
    • RGB - "rgb(100, 0, 127)"
    • HSL - "hsl(15, 82%, 56%)

Typography Basics and Text-Align:

'font-family' can be a single value or a comma-separated list of values that determin what font an element uses. Each font will fall into one of two categories, either a "font family name" like "Times new Roman" or a "generic family name" like serif.

If a browser cannot find or does not support the first font in a list, it will use the nect one, then the next one and so on until it findsa a supported and valid font. This is why it;s best practice to include a list ofvalues for this property, starting with the font you want to be used most and ending with a generic fon family as a fallback.

'font-size' sets the size of the font. When giving a value to this property, the value should not contain any whitespace, ex. 22px

'font-weight' sets the boldness of text. This will accept keyword or numeric values such as "bold" or a number from 1-1000. 700 is equivalent to bold. Numeric values will be in increments of 100 up to 900.

'text-align'will align text horizontally within an element, and you can use the common keywords you may have com across in word processors as the value for this property such as 'left', 'right','center'
